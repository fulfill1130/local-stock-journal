# Public Release Checklist

Use this checklist before pushing or packaging a public repository.

## Must Not Be Tracked

- OAuth credentials, API keys, tokens, passwords, private keys, or `.env` files.
- Real profile state, holdings, transactions, watchlists, notes, or dividend income records.
- SQLite runtime databases, quote caches, generated market data, or local profile stores.
- Broker PDFs, screenshots, statements, spreadsheets, OCR output, or uploaded documents.
- Private IP addresses, private hostnames, local-only launchers, or machine-specific paths.
- Logs, backups, generated reports, downloaded binaries, temporary files, or cache files.

## Required Public Files

- `LICENSE`
- `README.md`
- `docs/SECURITY_AND_PRIVACY.md`
- `docs/DISCLAIMER.md`
- Synthetic sample data only under `sample_data/`

## Verification Commands

```powershell
git status --short
git ls-files | rg -i "(data/|profiles/|uploads/|output/|backups/|config/|\.sqlite|\.db|credential|token|secret|\.env|\.pdf|\.png|\.jpe?g|\.xlsx?|\.docx?|\.log)"
python -m unittest discover -s tests -v
git diff --check
```

The `git ls-files` check should return no private runtime data. Review any match manually before publishing.
